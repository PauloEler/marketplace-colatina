# Guia de OKRs

## Objetivo

Usar objetivos e resultados-chave para ligar estratégia, execução e evidência.

## Regras

- objetivo descreve mudança relevante, não lista de tarefas;
- resultado-chave é verificável, possui fonte e prazo;
- entrega de funcionalidade não prova resultado por si só;
- metas sem linha de base devem começar pela medição;
- nenhum OKR autoriza desenvolvimento, gasto ou publicação;
- revisão mensal registra progresso, risco e decisão.

## Ciclo

1. Definir problema e linha de base.
2. Aprovar objetivo e KRs.
3. Vincular Sprints autorizadas.
4. Medir sem expor dados pessoais.
5. Revisar e registrar aprendizado.
